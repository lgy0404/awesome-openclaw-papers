Contributing Guide

Thank you for helping grow the OpenClaw research community! Adding a paper is as simple as editing a single YAML file.

1

Fork the Repo

Fork awesome-openclaw-papers on GitHub.

2

Edit the Data File

Add your entry to docs/_data/papers.yml — the single source of truth for all content.

3

Submit a PR

Open a Pull Request. The site will auto-rebuild once merged via GitHub Pages.

Data File Format

All papers, blog posts, and projects live in docs/_data/papers.yml. Here's the full schema:

Adding a Paper

Add your entry under the matching category in categories:

categories:
  - id: security-safety          # existing category
    papers:
      # ... existing papers ...

      - title: "Your Paper Title"
        authors: "Author A, Author B"     # optional
        description: >
          A 1-2 sentence summary of the paper's
          key contribution.
        date: "2026-03"                    # optional, YYYY-MM
        links:
          arxiv: https://arxiv.org/abs/XXXX.XXXXX
          pdf: https://example.com/paper.pdf   # optional
          code: https://github.com/...         # optional
          project: https://example.com         # optional
        tags: [security, agent, benchmark]     # optional

Adding a Blog Post

blog_posts:
  # ... existing posts ...

  - title: "Your Article Title"
    description: Brief description of the article.
    url: https://example.com/article
    tags: [architecture, tutorial]

Adding a Related Project

projects:
  # ... existing projects ...

  - name: ProjectName
    description: What the project does
    url: https://github.com/org/repo

Adding a New Category

If your paper doesn't fit an existing category, add a new one:

categories:
  # ... existing categories ...

  - id: your-category-id
    name: "Your Category Name"
    icon: "🔬"
    description: Brief description of this category
    papers:
      - title: "First Paper in This Category"
        description: >
          Description here.
        links:
          arxiv: https://arxiv.org/abs/XXXX.XXXXX

The website and filter buttons will automatically pick up the new category — no template changes needed.

Quality Checklist

  • Paper is directly related to OpenClaw or its ecosystem
  • Description is concise (1-2 sentences) and focuses on contribution
  • At least one working link is provided
  • Entry is placed in the correct category
  • YAML syntax is valid (indentation, quotes, etc.)
  • No duplicate entries

Don't Want to Edit YAML?

No problem! You can also submit a paper via GitHub Issue and a maintainer will add it for you.

Local Preview

To preview the site locally before submitting:

# Install Jekyll (one-time)
gem install bundler jekyll

# Run the site
cd docs
bundle exec jekyll serve

# Open http://localhost:4000
